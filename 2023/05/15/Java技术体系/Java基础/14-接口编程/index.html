1.接口

1.1 Java接口定义

在Java程序中，我们将使用关键字interface声明的特殊类称之为接口。
接口的定义和类的定义很相似，分为接口声明和接口体。

例如：

对于一个接口，我们要记住：

  • 接口用来描述类应该做什么，而不是指定它们具体应该如何做，也就是说，Java接口从技术层面来说不应该包含实现,
    而是定义一些抽象的方法或者常量。接口定义来就是要被其他类去实现的。
  • 一个类可以实现一个或多个接口。（下一节会介绍）
  • 接口不是普通的类，而是对希望符合这个接口的类的一组定义。
  • 接口中的所有方法都自动是public的（无须提供public关键字）。
  • 接口中可以定义常量。

注意：接口绝不会有实例字段，在java8之前，接口绝不会实现方法，
现在我们已经可以在接口中提供简单方法（但这些方法不能引用实例变量），
提供实例变量和方法实现的任务应该由实现接口的那个类来完成。 因此我们可以将接口看成是没有实例变量的抽象类。

4.接口回调

4.1 什么是接口回调

接口回调是指一个接口声明的变量被赋予实现类的对象地址时，可以调用一个类实现的接口方法，
此时这个变量也就是实现类的上转型对象,接口的这个特性是实现多态的关键点。
