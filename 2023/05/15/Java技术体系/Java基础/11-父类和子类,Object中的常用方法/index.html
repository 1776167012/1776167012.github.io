2.子类继承了父类的什么？

子类继承了父类一部分的变量和方法。

我们知道类的两个重要的成员：变量和方法。
父类有自己的变量和方法，子类继承了父类的一些成员，所以子类不仅拥有自己的变量和方法，而且子类也可以拥有父类定义的一些变量和方法。
子类可以直接使用它继承的父类中的成员和方法。

我们都知道类是通过它的构造方法来为自己分配变量内存空间的， 由于子类继承了父类，
所以子类的构造方法默认会先调用父类的构造方法，然后再用自己的构造法方法。（但这并不是继承，即子类不继承父类的构造方法）
继承很好的解决了代码复用以及功能纵向扩展。

3.继承的限制

子类并不能继承父类的所有变量和方法，它是有限制的。

  • 子类和父类在同一包中的继承性：除了private成员变量和方法，其他public,protected,无修饰符的变量和方法都继承了。
  • 子类和父类不在同一包中：子类只继承父类的protected和public成员。（这里protected和友好类的差别出现了）

5.1 protected Object clone()

5.1.1 方法介绍

创建并返回此对象的克隆副本。（副本和本体是两个独立的对象）
注意：这个方法是用protected修饰符修饰的。

用protected修饰的成员变量和方法被称为受保护的成员变量和方法。
当在另外一个类中用一个类创建了一个对象后，
如果两个类在同一包中，那么该对象可以访问该类的protected变量和方法，否则不可见。

所以，当我们在自己的包中新建一个类并使用Object创建对象的时候，我们是不能直接使用其clone方法的。
